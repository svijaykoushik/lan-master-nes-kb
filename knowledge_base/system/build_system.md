# Build Pipeline Overview

Lan Master utilizes a straightforward assembly-to-binary pipeline. The project is compiled using a batch script that invokes the NESASM assembler to transform 6502 assembly source code into a functional NES ROM file.

## Toolchain

The primary tool used for the build is **NESASM**. 

- **Input**: `game.asm` (The master assembly file).
- **Output**: `game.nes` (The final ROM binary).

## Compilation Process

The build process is automated via `compile.bat` with the following execution flow:

1. **Assembly**: The script calls `nesasm.exe game.asm`. The assembler processes `game.asm`, which includes other modules (such as `levels.asm`, `palette.asm`, etc.) and compiles them into the NES binary format.
2. **Cleanup**: The script deletes the `game.fns` file, which is a temporary file generated by NESASM containing symbol information.
3. **Execution**: The script triggers the launch of `game.nes`, typically using an external emulator, to allow the developer to immediately test changes.

## Entry Point

The `game.asm` file serves as the root of the project. It defines the memory map, includes all necessary source modules, and contains the reset vector that the NES hardware calls upon power-on.
